# NovelForge — 웹소설 기획 도우미 기획서

> 버전: 1.0 | 작성일: 2026-06-10
> 목적: 웹소설 작가가 아이디어를 투입하면 AI가 기획 문서를 구조화해주는 도구의 제품 기획

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | NovelForge (웹소설 기획 도우미) |
| 목적 | 작가의 거친 아이디어를 로그라인·시놉시스·플롯·트리트먼트·스토리 바이블로 각각 독립 구조화 |
| 핵심 원칙 | 집필은 작가, 구조 정리는 AI |
| 참고 모델 | Scrivener (기획·정리 방식), 한국 드라마 트리트먼트 체계 |
| 연동 도구 | Notion, Google Docs |
| 비용 정책 | 무료 사용 가능 |
| 대상 사용자 | 장기 연재(100화+)를 목표로 하는 웹소설 작가 |

## 2. 핵심 철학

**AI는 정리사, 작가는 창작자.**

이 도구는 소설을 쓰지 않는다. 작가가 던진 아이디어를 보기 좋은 표와 구조로 정돈해줄 뿐이다. 아이디어의 방향, 인물의 이름, 세계관의 규칙은 모두 작가가 제공한다. AI는 그것을 읽기 쉽게 배열하고 빠진 부분을 물어볼 뿐이다.

**모듈 독립성.**

로그라인을 만든다고 해서 시놉시스가 자동 생성되지 않는다. 작가가 원할 때, 원하는 문서만 따로 만든다. 아이디어의 완성도와 무관하게 필요한 문서부터 시작할 수 있다.

**장기 연재 우선.**

단편 완결이 아닌 100화 이상의 시리즈를 기준으로 설계한다. 복선이 얽히고 인물이 쌓이며 아크가 반복되는 환경에서도 흔들리지 않는 구조 관리가 목표다.

## 3. 문서 5종 — 정의, 역할, 분량

업계 실제 작성 순서: **로그라인 → 시놉시스 → 플롯 → 트리트먼트 → 집필**
이 도구는 집필 직전 단계까지만 지원한다.

| 문서 | 분량 기준 | 역할 | 작성 목적 |
|------|----------|------|----------|
| 로그라인 | 1~2문장 | 작품의 핵심 한 줄 압축 | 아이디어 자가 검증, 플랫폼 피칭 |
| 시놉시스 | 1~15페이지 | 전체 이야기 뼈대 (결말 포함) | 편집자 제출, 전체 방향 설정 |
| 플롯 | 표 형식 | 회차 단위 감정 흐름·갈등·장면 | 연재 전 호흡 조율 |
| 트리트먼트 | 회차 카드 | 장면별 세부 (시간·장소·감정·복선) | 집필 전 머릿속 그림 고정 |
| 스토리 바이블 | 참조 문서 | 인물·세계관·용어 설정 고정 | 장기 연재 일관성 유지 |

## 4. 화면 구성 컨셉 (Scrivener 기반)

Scrivener의 핵심 구조인 **바인더(Binder) + 편집 공간 + 메타데이터 패널** 세 영역을 웹소설 기획에 맞게 재해석한다.

```
┌─────────────────────────────────────────────────────────┐
│  [작품 선택]  NovelForge                                  │
├──────────────┬──────────────────────────┬───────────────┤
│              │                          │               │
│  [바인더]    │   [편집·미리보기 공간]   │  [메모 패널]  │
│              │                          │               │
│  📁 작품명   │  현재 선택한 문서의      │  작가 메모    │
│   ├ 로그라인 │  내용을 표·카드 형태로   │  복선 메모    │
│   ├ 시놉시스 │  표시. AI 생성 결과가    │  핵심 대사    │
│   ├ 플롯     │  여기 출력됨.            │  다음 할 일   │
│   ├ 트리트먼트│                         │               │
│   │  ├ 1화   │                          │               │
│   │  ├ 2화   │                          │               │
│   │  └ ...   │                          │               │
│   └ 스토리   │                          │               │
│     바이블   │                          │               │
│              │                          │               │
│  [+ 새 작품] │  [AI 생성] [Notion 저장] │               │
│              │  [Docs 내보내기]         │               │
└──────────────┴──────────────────────────┴───────────────┘
```

- **바인더:** 작품별·문서 종류별 계층 구조. 트리트먼트는 회차별로 하위 항목 펼침.
- **편집 공간:** AI가 생성한 표·카드가 여기 표시됨. 작가가 직접 수정 가능.
- **메모 패널:** 집필 중 떠오른 이미지·대사·다음 화 구상을 빠르게 메모.

## 5. AI 상호작용 모델

### 아이디어 입력 → AI 확인 질문 → 문서 선택 → 생성

작가는 아이디어를 자유 형식으로 입력한다. 문법도, 완성도도 상관없다.

**입력 예시:**
> "재벌 비서로 일하다가 억울하게 죽은 여자가 10년 전으로 돌아가서 자기를 버린 재벌 집안에 복수하는 이야기인데, 근데 막상 돌아와보니 자기를 죽인 게 재벌 회장 아들이 아니라 회사 임원 중 한 명이었다는 걸 알게 되는 거"

**AI 확인 질문 (2~3개, 필수 최소화):**

| 질문 목적 | 예시 질문 |
|---------|---------|
| 장르·톤 확인 | "이 이야기의 분위기는 통쾌한 복수극에 가까운가요, 아니면 서스펜스·스릴러에 가까운가요?" |
| 결말 방향 확인 | "주인공이 결국 복수에 성공하는 결말인가요, 아니면 다른 방향을 생각하고 계신가요?" |
| 로맨스 비중 확인 | "로맨스 라인이 있다면 주인공과 누구 사이인가요?" |

작가가 "모르겠다"고 해도 괜찮다. AI는 미확인 항목을 빈 칸으로 남겨두고 나머지를 먼저 구조화한다.

**문서 선택 (독립 선택):**

```
어떤 문서를 만들까요? (복수 선택 가능, 동시 생성 아님)

  ☐ 로그라인     — 아이디어를 1~2문장으로 압축
  ☐ 시놉시스     — 전체 이야기 5단 구조표
  ☐ 플롯         — 10화 블록 단위 감정흐름표
  ☐ 트리트먼트   — 회차 카드 (장면·감정·복선)
  ☐ 스토리 바이블 — 인물·세계관 설정집
```

선택한 문서만 생성. 나머지는 나중에 별도로 만들 수 있다.

## 6. 로그라인 구조

**역할:** 아이디어를 딱 한 줄로 검증하는 관문. 로그라인이 설득력 없으면 시놉시스도 흔들린다.

### 입력 → 출력 구조

| 항목 | 내용 | 예시 |
|------|------|------|
| 주인공 | 어떤 인물인가 (한 단어) | 재벌가 비서 |
| 출발 사건 | 무엇이 달라졌나 | 10년 전으로 회귀 |
| 핵심 목표 | 반드시 이루어야 하는 것 | 자신을 죽인 진범 색출 및 복수 |
| 핵심 장애 | 목표를 막는 가장 큰 벽 | 재벌 권력망 안에서 진실 접근 불가 |
| 차별 요소 | 이 작품만의 독특한 점 | 예상 복수 대상이 아닌 숨겨진 진범 |
| **완성 로그라인** | AI가 위 5개를 1~2문장 조합 | *"억울한 죽음 뒤 10년 전으로 돌아온 재벌 비서가, 자신을 죽인 진짜 범인이 예상과 다름을 깨닫고 재벌 권력망 안에서 진실을 파헤치는 이야기"* |

## 7. 시놉시스 구조

**역할:** 작품 전체의 뼈대. 결말까지 포함한 완전한 이야기 지도.
카카오·네이버 투고 시놉시스는 캐릭터 소개보다 **플롯의 흡인력**을 우선시하며, 주인공의 최종 목표 달성 여부가 반드시 포함되어야 한다.

### 출력 형태

| 구분 | 내용 |
|------|------|
| 장르·분위기 키워드 | 3개 이내 (예: 현대 / 회귀 / 통쾌한 복수극) |
| 주인공 핵심 욕망 | "~해야 한다" 한 문장 |
| 발단 | 사건의 시작, 주인공이 놓인 상황 |
| 전개 | 갈등의 확대, 관계와 정보의 변화 |
| 위기 | 최대 위협, 포기 직전 순간 |
| 절정 | 결정적 선택 또는 대결 |
| 결말 | 주인공이 얻거나 잃는 것, 세계의 변화 |
| 핵심 주제 | 작품이 말하고자 하는 것 (한 문장) |

## 8. 플롯 구조

**역할:** 회차 단위로 독자의 감정을 어떻게 끌고 갈 것인지 설계하는 흐름표.

### 업계 기준

| 항목 | 기준 |
|------|------|
| 회차 분량 | 5,000~5,500자 (공백 포함), 스마트폰 10~15분 독서 |
| 연재 주기 | 주 5~7화 (주당 약 25,000~38,500자) |
| 아크 단위 | 미니아크 2~5화 / 중간아크 10~50화 / 마무리 권장 15~20화 이내 |

### 출력 형태 — 10화 블록 단위

| 회차 | 감정 흐름 | 주요 갈등 | 핵심 장면 요약 | 클리프행어 유형 |
|------|-----------|-----------|---------------|----------------|
| 1화 | 호기심·낯섦 | — | 주인공 소개, 회귀 순간 | 충격 발언형 |
| 2화 | 긴장 고조 | 내적 갈등 | 과거와 다른 점 첫 발견 | 위기 직전 컷 |
| 3화 | 경계·탐색 | 정보 갈등 | 진범 단서 첫 포착 | 비밀 발각형 |
| … | … | … | … | … |
| 10화 | 카타르시스 | 외적 갈등 최고조 | 1막 클라이맥스 | 인물 등장형 |

10화 블록이 하나의 **아크(Arc)**. 아크가 끝나면 다음 10화 블록을 새로 생성.

### 5가지 클리프행어 유형

| 유형 | 설명 |
|------|------|
| 위기 직전 컷 | 위험·대결 직전에 화 종료 |
| 충격 발언형 | 임팩트 있는 대사·폭로 직후 종료 |
| 인물 등장형 | 핵심 인물의 예상치 못한 등장으로 종료 |
| 위험 신호형 | 위험이 감지되는 순간에 종료 |
| 비밀 발각형 | 숨겨진 진실이 드러나기 직전에 종료 |

### 아크 설계 원칙

- **감정 호흡 사이클:** 긴장 3화 → 이완·웃음 1화 → 긴장 재상승 반복
- **10화 법칙:** 10화 이내에 세계관·주인공·핵심 갈등 완전 제시
- **아크 완결:** 각 아크는 독립적인 소만족을 주되 다음 아크를 기대하게 한다

## 9. 트리트먼트 구조

**역할:** 회차 안을 장면 단위로 쪼개어 집필 전 머릿속 그림을 고정하는 메모 카드.

한국 드라마 트리트먼트는 "그림 없는 콘티"라 불린다. 장소·타이밍·감정 심리를 구체화한 산문 형식으로, 대사는 초안 수준만 메모하고 기술적 지시는 포함하지 않는다. 웹소설도 동일하게 적용하면 100화 이상 연재 시 플롯 꼬임을 방지할 수 있다.

### 회차 카드 구조 (1화 = 1카드)

#### 기본 정보
| 항목 | 내용 |
|------|------|
| 회차 번호 | |
| 시간대 | 낮 / 밤 / 새벽 / 황혼 등 |
| 장소 | 구체적 공간 (예: 비가 내리는 회사 주차장 옥상) |
| 분위기 키워드 | 3개 이내 (예: 긴장·고요·절망) |
| 이 화의 목적 | 독자에게 무엇을 느끼게 할 것인가 |

#### 장면 흐름
| 순서 | 장면 요약 | 등장인물 | 감정 변화 (시작 → 끝) |
|------|-----------|---------|---------------------|
| 장면 1 | | | |
| 장면 2 | | | |
| 장면 3 | | | |

#### 복선 메모
| 복선 내용 | 이 화에서 처리 방식 | 회수 예정 시점 |
|---------|-------------------|--------------|
| | | |

#### 작가 메모 (자유 형식)
- **핵심 이미지:** 집필 중 떠오른 시각적 장면·분위기 스케치
- **핵심 대사 초안:** 확정 아님, 톤·분위기 참고용
- **기타:** 독자 반응 예측, 다음 화와의 연결 고리

## 10. 스토리 바이블 (Story Bible)

**역할:** 장기 연재 시 인물·세계관·용어가 회차별로 흔들리지 않도록 고정하는 참조 문서.
시놉시스·플롯·트리트먼트와 독립된 별도 섹션. 연재 시작 전 초안 작성 후 연재 중 지속 업데이트.

### 인물 설정집

| 항목 | 내용 |
|------|------|
| 이름 (본명·별칭) | |
| 첫 등장 회차 | |
| 역할 | 주인공 / 조력자 / 빌런 / 조연 |
| 핵심 욕망 | 이 인물이 진짜 원하는 것 |
| 핵심 결핍 | 내면의 상처 또는 약점 |
| 말투·어조 특징 | 대사만 보고 알아볼 수 있는 특성 |
| 주요 관계 | 다른 인물과의 관계 요약 |
| 성장 궤적 | 시작 상태 → 결말 상태 |

### 세계관 설정집

| 항목 | 내용 |
|------|------|
| 세계의 규칙 | 이 세계에서만 작동하는 핵심 법칙 |
| 파워 시스템 | 능력·등급·성장 방식 |
| 주요 장소 | 자주 등장하는 공간과 분위기 메모 |
| 시대·역사 배경 | 연대기 또는 중요 과거 사건 |
| 용어·고유명사 사전 | 독자에게 설명이 필요한 단어 목록 |

### 문서 간 연결 구조

```
스토리 바이블 (인물 설정집)
    ↕ 연동
트리트먼트 카드 (등장인물 항목)
    ↕ 확인
복선 트래커 (전체 작품 복선 목록)
    ↕ 참조
플롯 표 (해당 회차 갈등·장면)
```

트리트먼트 카드에서 인물을 선택하면 인물 설정집의 해당 항목을 바로 참조할 수 있다.
복선 트래커에서 특정 회차를 클릭하면 해당 트리트먼트 카드로 이동한다.

## 11. AI 성공 패턴 체크리스트

작가가 문서를 생성한 후, AI가 100만 독자 달성 웹소설의 공통 패턴과 비교해 빠진 부분을 체크리스트로 출력한다. 작가가 따를 의무는 없다. 참고용.

### 공통 체크리스트 (장르 무관)

| 항목 | 확인 질문 |
|------|----------|
| 1화 후킹 | 3페이지 이내에 독자를 붙잡는 사건 또는 반전이 있는가? |
| 주인공 차별점 | "이 주인공만 할 수 있는 것"이 1문장으로 정의되는가? |
| 회차 완결감 | 매 화에 독립적인 작은 카타르시스가 있는가? |
| 성장 지표 | 독자가 성장을 체감할 수 있는 시각적 수단이 있는가? |
| 10화 법칙 | 10화 이내에 세계관·주인공·핵심 갈등이 모두 제시되는가? |
| 빌런 납득성 | 빌런의 논리가 독자가 이해할 수 있는 수준인가? |
| 감정 호흡 | 긴장-이완 사이클이 설계되어 있는가? |

### 장르별 추가 체크 (해당 장르 선택 시 보조 표시)

**회귀물:** 회귀 동기 납득성 / 전생 지식 활용 속도 / 과거-미래 복선 대칭
**로맨스판타지:** 20화 이내 감정선 암시 / 50화 이내 첫 설렘 장면 / 여주 독립 목표
**헌터물:** 시스템 독창성 / 스케일 상승 로드맵 / 성장 수치 가시화

## 12. 사용 흐름 (User Flow)

```
① 아이디어 입력
   작가가 자유 형식으로 줄거리·분위기·장르를 입력
   (두리뭉술해도, 미완성이어도 괜찮음)
        ↓
② AI 확인 질문 (2~3개)
   장르·톤·결말 방향 등 최소한의 방향 확인
   "모르겠다" 가능 → 미확인 항목은 빈 칸으로 처리
        ↓
③ 문서 선택 (독립 선택)
   로그라인 / 시놉시스 / 플롯 / 트리트먼트 / 스토리 바이블
   원하는 것 하나만 선택해도 됨
        ↓
④ AI 구조화 생성
   선택한 문서의 형식에 맞춰 표·카드 출력
   작가가 제공한 정보만 사용, AI가 내용 창작하지 않음
        ↓
⑤ 작가 수정·보완
   편집 공간에서 직접 수정 가능
        ↓
⑥ 체크리스트 확인 (선택)
   성공 패턴 대비 피드백 열람 (선택 사항)
        ↓
⑦ 저장 및 내보내기
   Notion 자동 저장 또는 Google Docs 내보내기
        ↓
⑧ 장기 연재 관리
   아크 진행 현황, 복선 트래커, 연재 캘린더 업데이트
```

## 13. 외부 도구 연동

### Notion 연동 구조

| Notion 뷰 | 연결 문서 | 활용 방식 |
|-----------|---------|---------|
| 테이블뷰 | 작품 목록 + 시놉시스 요약 | 여러 작품 한눈에 관리 |
| 보드뷰 | 플롯 아크별 카드 | 진행중/완료 상태 구분 |
| 페이지 (1:1) | 트리트먼트 회차 카드 | 회차별 상세 내용 |
| 데이터베이스 | 복선 트래커 | 회수 여부 체크박스 포함 |
| 캘린더뷰 | 연재 예정 회차 + 날짜 | 집필 일정 시각화 |
| 관계형 DB | 인물 ↔ 회차 ↔ 감정선 | 인물 설정집 연동 |

Notion 무료 플랜 기준으로 전체 기능 사용 가능하도록 설계.

### Google Docs 연동

| 기능 | 내용 |
|------|------|
| 시놉시스 내보내기 | 편집자 제출용 문서 형식 |
| 플롯 표 내보내기 | 테이블 포함 Google Docs 문서 |
| 공유 | 편집자·파트너와 링크 공유 |

Google Docs 무료 계정 기준 지원.

## 14. 장기 연재 관리

나 혼자만 레벨업(22아크), 재벌집 막내아들(326화) 같은 장기 연재에서 핵심은 **아크 단위 관리**와 **복선 추적**이다.

| 관리 항목 | 내용 |
|---------|------|
| 작품 아카이브 | 작품별 폴더, 문서별 버전 이력 |
| 복선 트래커 | 설정 화수 + 회수 화수 + 회수 여부 체크박스 |
| 인물 관계 메모 | 첫 등장 화수, 관계 변화 타임라인 |
| 아크 진행 현황 | 현재 아크 완료율, 다음 아크 예정 내용 |
| 감정 곡선 기록 | 아크별 긴장-이완 사이클 메모 |
| 연재 캘린더 | 집필 예정 화수와 날짜 Notion 캘린더 연동 |

## 15. 이 도구가 하지 않는 것

- 소설 본문을 대신 집필하지 않는다
- 캐릭터 이름·세계관 설정을 AI가 창작해서 제안하지 않는다 (작가가 준 것만 정리)
- 작가의 아이디어를 수정하거나 방향을 바꾸도록 강요하지 않는다
- 로그라인을 만든다고 시놉시스가 자동 생성되지 않는다 (각 문서 독립 생성)
- 플랫폼별 투고 규정 검토는 이 도구의 범위 밖이다
