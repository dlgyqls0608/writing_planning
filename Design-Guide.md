## 1. 디자인 철학

NovelForge의 UI는 세 가지 원칙을 바탕으로 설계되었다.

| 원칙 | 설명 |
|------|------|
| **작가 몰입 우선** | UI 크롬은 최소화한다. 작가의 아이디어와 문서가 화면의 중심을 차지한다. |
| **투명한 AI 협력** | AI가 생성한 콘텐츠는 항상 명확히 표시되고, 작가가 언제든 수정·삭제할 수 있다. AI는 강요하지 않는다. |
| **Scrivener식 구조화** | 3패널 레이아웃(바인더 · 편집 공간 · 메모)으로 전체 구조와 세부 내용을 동시에 파악한다. |

## 2. 색상 시스템

### 2-1. 디자인 토큰 3계층 구조

업계 표준(IBM Carbon, Google Material Design)에 따라 토큰을 3단계로 분리한다.

```
참조 토큰 (Raw)       시맨틱 토큰 (의미)       컴포넌트 토큰 (용도)
──────────────────    ─────────────────────    ──────────────────────
#7c3aed          →   --c-logline          →   .b-logline, .tab-logline
#4f46e5          →   --c-accent           →   .btn-primary 배경, input:focus 테두리
#1e1e2e          →   --c-text             →   body 텍스트, 제목
#eef0f5          →   --c-bg               →   앱 전체 배경
```

> 구현 시 CSS Custom Properties(변수)로 선언하며, 컴포넌트는 반드시 시맨틱 토큰을 참조한다.
> 원시 hex 값을 컴포넌트 스타일에 직접 기재하지 않는다.

---

### 2-2. 문서 타입 색상

각 문서 유형에는 고유한 색상 정체성이 부여된다. 배지, 탭, 강조 상자에 일관되게 적용한다.

| 토큰 | HEX | 밝은 배경(Tint) | 문서 유형 |
|------|-----|----------------|----------|
| `--c-logline`    | `#7c3aed` | `#ede9fe` | 로그라인 (보라) |
| `--c-synopsis`   | `#2563eb` | `#dbeafe` | 시놉시스 (파랑) |
| `--c-plot`       | `#059669` | `#d1fae5` | 플롯 (초록) |
| `--c-treatment`  | `#d97706` | `#fef3c7` | 트리트먼트 (황색) |
| `--c-bible`      | `#0891b2` | `#cffafe` | 스토리 바이블 (청록) |
| `--c-foreshadow` | `#dc2626` | `#fee2e2` | 복선 트래커 (빨강) |

**배지 사용 규칙**

- 배경: Tint 색상 적용
- 텍스트: 해당 토큰 색상(진한 값) 적용
- 예) 로그라인 배지: `background: #ede9fe; color: #7c3aed`

### 2-5. 접근성 (WCAG 2.2)

모든 색상 조합은 WCAG 2.2 Level AA 기준을 충족해야 한다.

| 요소 유형 | 최소 대비율 | 기준 |
|-----------|------------|------|
| 일반 텍스트 (18px 미만) | **4.5 : 1** | WCAG 1.4.3 |
| 큰 텍스트 (18px 이상 또는 14px bold) | **3 : 1** | WCAG 1.4.3 |
| UI 컴포넌트 (버튼 테두리, 아이콘) | **3 : 1** | WCAG 1.4.11 |

**주의 항목** — 다크 사이드바에서 반드시 검증할 조합:

| 텍스트 | 배경 | 검증 필요 이유 |
|--------|------|---------------|
| `#9aa0c0` | `#18182a` | 일반 항목 텍스트 — 4.5:1 충족 여부 확인 |
| `#4b5090` | `#18182a` | 섹션 레이블 — 장식 목적이면 예외 허용 가능 |

권장 검증 도구: [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) · Chrome DevTools 접근성 패널 · axe DevTools

```
┌──────────────┬─────────────────────────┬──────────────┐
│   바인더      │      편집 공간           │   메모 패널   │
│  (215px)     │      (flex: 1)          │   (215px)    │

│  border-right│                         │  border-left │
```

## 6. 컴포넌트 라이브러리

|------|------|----------|--------|------------|

### 6-12. 바인더 항목 (Binder Item)

3패널 레이아웃의 좌측 바인더 내 문서 트리 항목.

| 상태 | 배경 | 텍스트 색 | Left border |
|------|------|----------|-------------|
| 기본 | transparent | `#6b7280` | 없음 |
| Hover | `#ecedf5` | `#6b7280` | 없음 |
| 활성 | `#eef0ff` | `#4f46e5` | 3px `#4f46e5` |

- 작품(루트): padding-left 14px, font-weight 700
- 문서: padding-left 26px
- 서브 항목(화차 등): padding-left 40px, font-size 11px, color `#9ca3af`

### 6-13. 빈 상태 (Empty State)

작가가 아직 콘텐츠를 만들지 않은 화면에서 다음 행동을 유도한다.

| 유형 | 설명 | 적용 화면 |
|------|------|----------|
| **유도형** | 안내 텍스트 + Primary 버튼 CTA | S-01 첫 방문 |
| **예시 데이터형** | 샘플 작품 카드 1개를 미리 노출해 UI 구조 파악 지원 | S-01 튜토리얼 |
| **AI 제안형** | "어떤 문서부터 시작할지 모르겠다면?" 추천 배너 | S-05 첫 생성 전 |
| **통계형** | 항목 수 0개 표시 + `[+ 추가]` 버튼 | S-11 복선 없을 때 |

빈 상태 텍스트는 부드럽고 격려하는 어조를 유지한다. 오류처럼 보이는 경고 문구는 사용하지 않는다.

## 8. AI 인터랙션 패턴

### 8-1. 인라인 편집 (Inline Edit)

테이블 셀 또는 텍스트 블록을 클릭해 바로 수정한다.

```
트리거:  셀 클릭
전환:    border-color → #818cf8, background → #fafbff
컨트롤:  [확인 ✓] (Primary·sm) / [취소 ✗] (Ghost·sm) 인라인 표시
적용:    S-06~S-10 모든 결과 화면 테이블 셀 및 완성 텍스트 박스
```

---

### 8-2. AI 재생성 (Regenerate)

기존 AI 결과물을 전부 교체한다.

```
1. [AI 재생성] 버튼 클릭
2. 확인 팝업: "현재 내용이 삭제됩니다. 계속할까요?" — [취소] [계속]
3. 로딩 중: 버튼 비활성화 + 인라인 스피너
4. 완료 후: 변경된 항목 highlight (flash 애니메이션 0.5s)
```

---

### 8-3. 수락 / 거절 패턴 (Accept · Reject)

2025년 AI UX 표준. AI가 새 내용을 제안할 때 기존 내용과 나란히 diff 뷰로 표시한다.

```
기존 내용 (취소선 · 빨간 배경)  →  새 제안 (초록 배경)

[수락 ✓]  (Primary·sm)  — 새 내용으로 교체
[거절 ✗]  (Danger·sm)   — 기존 내용 유지
[수동 수정] (Ghost·sm)  — 편집기에서 직접 조정

적용: AI 재생성 완료 후 / S-12 AI 제안 수락 시
```

---

### 8-4. Refine Output (세부 조정)

생성된 텍스트를 선택하면 컨텍스트 메뉴가 나타난다.

```
선택 가능 옵션: [더 짧게] [더 강하게] [더 부드럽게] [다시 쓰기]
결과:           선택한 텍스트 범위만 교체, Accept·Reject 패턴 연결
적용:           S-06 로그라인, S-07 시놉시스 텍스트 선택 시
```

---

### 8-5. AI 투명성 원칙

| 규칙 | 세부 내용 |
|------|---------|
| **AI 생성 표시** | AI가 만든 콘텐츠에는 ✨ 아이콘 + "AI 생성" 레이블 표시 |
| **강요 없음** | 모든 AI 결과는 수정·삭제·무시 가능. 반드시 따를 필요 없음 명시 |
| **모르겠다 허용** | S-04 질문에서 "모르겠다" 선택 시 해당 항목 빈 칸으로 두고 진행 |
| **제안 구분** | S-12 체크리스트 AI 제안은 "참고용"임을 문구로 명시 |

---

### 8-6. 빈 상태 온보딩 (Empty State Onboarding)

첫 방문 혹은 콘텐츠가 없을 때 작가가 막히지 않도록 안내한다.

| 화면 | 온보딩 방식 |
|------|-----------|
| S-01 첫 방문 | 샘플 작품 카드 1개 미리 노출 (튜토리얼 레이블 포함) + `[새 작품 시작]` Primary CTA |
| S-05 첫 생성 전 | "어떤 문서부터 시작할지 모르겠다면?" AI 추천 배너 (로그라인 권장) |
| S-11 복선 없을 때 | 통계 0개 표시 + `[+ 첫 복선 추가]` 버튼 |

## 9. 접근성 체크리스트

구현 완료 후 배포 전 다음 항목을 검증한다.

### 색상 대비

- [ ] 일반 텍스트 (`--c-text` on `--c-bg`) ≥ 4.5:1 확인

- [ ] 아이콘 및 UI 컴포넌트 테두리 ≥ 3:1 확인

### 키보드 및 포커스

- [ ] 모든 인터랙티브 요소 Tab 키 접근 가능
